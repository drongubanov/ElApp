# ElApp

Веб-приложение для расчёта мощности и тока в сетях постоянного тока,
однофазных и трёхфазных сетях переменного тока. Работает полностью в
браузере, без сервера и без сторонних зависимостей.

## Возможности

- Расчёт активной (P), полной (S) и реактивной (Q) мощности, тока и
  напряжения для DC, однофазной и трёхфазной (симметричной) сети.
- Ввод любой из двух связанных величин (мощность или ток) — остальные
  вычисляются автоматически.
- Подбор рекомендуемого номинала автоматического выключателя по
  расчётному току.
- Подбор рекомендуемого сечения кабеля (медь/алюминий) по справочной
  таблице допустимых токовых нагрузок (приближена к ПУЭ гл. 1.3) с учётом
  способа прокладки (воздух / труба или кабель-канал / лоток-пучок) и
  числа кабелей, проложенных рядом.
- Расчёт потери напряжения на линии по длине кабеля и сечению
  рекомендованной жилы, со сравнением с общепринятой нормой (≤5%).
- История расчётов (хранится локально в `localStorage`), с возможностью
  повторно открыть или удалить запись.
- **Конструктор сети** — построение электрической схемы в виде дерева
  узлов: графический редактор с кликабельными блоками, соединёнными
  линиями (кабелями), в духе диаграммы иерархии проекта. Корневой узел —
  точка ввода (главный щит), у него могут быть дочерние узлы — следующие
  щиты или потребители, и так до любой глубины. Узел может одновременно
  иметь собственную нагрузку (тип сети, напряжение, cosφ, мощность или
  ток) и дочерние узлы — например, щит, часть нагрузки которого
  подключена напрямую, а часть — через под-щиты. На самом блоке есть
  кнопки ⚙ (открыть панель свойств), + (добавить дочерний узел) и −
  (удалить узел, с подтверждением, если у него есть дочерние узлы) — они
  работают независимо от того, какой узел выбран сейчас. В панели
  свойств задаются параметры узла и линии, питающей его от родителя
  (способ прокладки, число кабелей рядом, длина), а также коэффициент
  одновременности (Кс) для суммирования нагрузки его дочерних узлов. По
  кнопке
  «Рассчитать сеть» для каждого узла дерева — от листьев к корню —
  считается суммарная нагрузка, подбираются автоматический выключатель и
  кабель линии, считается падение напряжения на ней; результат и
  рекомендуемый автомат отображаются прямо на блоке и подробно — в панели
  свойств. Ошибка в одном узле блокирует расчёт только его самого и его
  предков, не затрагивая соседние ветви дерева. Схема сети сохраняется
  локально в `localStorage` и восстанавливается при следующем открытии
  страницы. При наведении на блок подсвечивается путь питания до корня;
  кнопкой «Сбросить сеть» схему можно очистить и начать заново. Кнопкой
  «Теплокарта» включается постоянная раскраска блоков и линий по
  расчётному току узла (от холодно-синего к горячему красному) — над
  деревом при этом появляется легенда шкалы с минимальным и максимальным
  током дерева, поясняющая цвета. Над деревом есть кнопки масштаба (40–200%,
  с шагом 10%) — удобно для крупных схем; перемещаться по холсту можно,
  зажав левую кнопку мыши на свободном месте (вне блоков) и потянув.
- **Экспорт однолинейной схемы** — собранную сеть можно сохранить
  отдельным файлом в виде оформленного чертежа: блоки щитов и нагрузок,
  соединённые линиями с подписями номинала автомата и сечения кабеля,
  на листе стандартного формата (A4…A1, ГОСТ 2.301) с рамкой и упрощённой
  основной надписью (ГОСТ 2.104). Доступны два формата:
  - **PDF** — готовый к печати лист (геометрия и подписи отрисовываются
    на холсте и вкладываются в одностраничный PDF; кириллица выводится
    системным шрифтом);
  - **DXF** (AutoCAD R12 / AC1009) — открытый векторный формат обмена,
    который AutoCAD и другие CAD-программы открывают как обычный чертёж и
    при необходимости пересохраняют в DWG. Формат DWG напрямую не
    поддерживается: это закрытый бинарный формат Autodesk без открытой
    спецификации, и сгенерировать его без сервера и сторонних библиотек
    нельзя; DXF — стандартная векторная альтернатива с тем же содержимым.

  > Блоки узлов изображаются прямоугольниками с поясняющими надписями, а
  > не полным набором условных графических обозначений (УГО) по
  > ГОСТ 2.755/21.614, а основная надпись приведена в упрощённом виде —
  > для итогового проекта чертёж следует доработать в CAD.

> Подбор автоматического выключателя и сечения кабеля носит справочный,
> ориентировочный характер. Расчёт ориентирован на действующую редакцию
> ПУЭ (гл. 1.3, 7-е издание) — таблицы 6-го издания устарели и для
> текущего проектирования не применяются, поэтому отдельное сравнение
> редакций в калькуляторе не приводится. Поправочные коэффициенты на
> способ прокладки и число кабелей — приближённые. Для проектирования
> реальных электроустановок используйте актуальные таблицы ПУЭ/ГОСТ с
> учётом температуры окружающей среды и реальных условий прокладки, либо
> обратитесь к специалисту.

## Запуск

Приложение — статический сайт без сборки. Достаточно открыть
`index.html` в браузере или поднять любой статический HTTP-сервер,
например:

```bash
npx serve .
# или
python3 -m http.server 8000
```

и открыть `http://localhost:<порт>/`.

## Используемые формулы

- **Постоянный ток:** `P = U · I`
- **Однофазная сеть переменного тока:** `S = U·I`, `P = U·I·cosφ`, `Q = √(S²−P²)`
- **Трёхфазная сеть (симметричная нагрузка):** `S = √3·U·I`, `P = √3·U·I·cosφ`, `Q = √(S²−P²)`,
  где `U` и `I` — линейное напряжение и линейный ток.
- **Потеря напряжения на линии:** `ΔU = k·ρ·L·I·cosφ/S`, где `k = 2` для DC и однофазной сети,
  `k = √3` для трёхфазной; `ρ` — удельное сопротивление жилы (медь 0.0175, алюминий 0.028 Ом·мм²/м);
  индуктивное сопротивление не учитывается (приближение для предварительной оценки).
- **Конструктор сети — суммарная потеря напряжения:** для каждого узла дерева
  складываются `dropPercent` всех линий по пути от точки ввода (корня) до этого
  узла — именно суммарную потерю у самого удалённого потребителя сравнивают с
  нормой ≤5%, а не потерю на одном участке (она может быть в норме, а итоговая —
  превышать норму). Суммирование процентов, а не вольт, — приближение, которое
  позволяет складывать потери на участках сети с разным напряжением.
- **Конструктор сети — термическая стойкость кабеля при КЗ:** минимально
  допустимое по нагреву сечение жилы `Sмин = Iкз·√t / k` (IEC 60364-4-43,
  ГОСТ Р 50571.4.43), где `Iкз` — трёхфазный ток КЗ в точке (наибольший из
  возможных, накопленный по сопротивлению кабелей от трансформатора), `t` —
  принятое время отключения защитой (0.1 с, если расчёт показывает гарантированное
  мгновенное срабатывание электромагнитного расцепителя, иначе консервативно 5 с —
  нормативный максимум для систем TN по ПУЭ-7 п. 1.7.79), `k` — коэффициент жилы
  с ПВХ-изоляцией (медь 115, алюминий 76). Полученное `Sмин` сравнивается с фактически
  подобранным по нагрузке сечением — иначе выбранный по току нагрузки тонкий кабель может
  не выдержать тепловой импульс короткого замыкания, особенно вблизи мощного источника.
- **Конструктор сети — поправка на температуру среды и тип изоляции:** табличный
  допустимый ток жилы дан для базовой температуры воздуха +25 °C; при иной
  температуре он умножается на коэффициент `k = √((θmax − θсреды) / (θmax − 25))`
  (ГОСТ Р МЭК 60364-5-52, ПУЭ табл. 1.3.3), где `θmax` — максимально допустимая
  температура жилы по типу изоляции (ПВХ — 70 °C, сшитый полиэтилен/СПЭ — 90 °C).
  Этот коэффициент перемножается с поправками на способ прокладки и число кабелей
  рядом; изоляция с более высокой `θmax` снижает ток медленнее при нагреве среды.
  Если температура среды не ниже `θmax`, длительная работа кабеля невозможна и
  сечение не подбирается.
- **Конструктор сети — петля «фаза–защитный проводник» и система заземления:**
  однофазный ток КЗ `Iкз(1) = Uф / (Zт + Rфаз + Rpe)` считается по фактическому
  сечению защитного проводника (PE/PEN по ПУЭ-7 табл. 1.7.5), которое может быть
  меньше фазного, — поэтому петля имеет сопротивление больше `2·Rфаз`, а ток
  ниже (для сечений ≤16 мм² PE равно фазному и формула совпадает с прежней
  `Uф/(Zт + 2·Rфаз)`). Система заземления точки ввода (TN-C-S / TN-S / TT)
  определяет, чем обеспечивается автоматическое отключение при замыкании на
  корпус: в системах TN — максимально-токовой защитой по этой петле, в системе
  TT ток замыкания на землю ограничен сопротивлением заземлителей, поэтому
  отключение должно обеспечиваться УЗО (RCD) — об этом выводится отдельное
  замечание.
- **Конструктор сети — распределение по фазам и ток нейтрали:** для трёхфазного
  узла можно задать доли нагрузки по фазам L1/L2/L3 (любые относительные веса,
  нормируются автоматически). Ток фазы `Iк = 3·sк·I`, где `sк` — доля фазы, а
  `I` — симметричный линейный ток узла; ток в нулевом рабочем проводнике от
  несимметрии `Iн = √(I1² + I2² + I3² − I1·I2 − I2·I3 − I3·I1)` (по основной
  гармонике, без учёта высших гармоник). При равных долях нейтраль равна нулю,
  при нагрузке на одной фазе — её ток. Если самая загруженная фаза превышает
  номинал автомата, подобранного по симметричному току, выводится замечание о
  перекосе фаз.
- **Конструктор сети (узел дерева):** активная и реактивная мощности узла
  складываются из его собственной нагрузки и суммы P/Q всех его дочерних узлов
  (уже рассчитанных рекурсивно, от листьев к корню), с учётом коэффициента
  одновременности (`P = Кс·(Pсобств + ΣPдетей)`, аналогично для Q), затем
  `S = √(P²+Q²)` и ток узла считается по тем же формулам, что и для одного
  блока. Это упрощение (фазы и напряжения дочерних узлов не приводятся к
  общему вектору) — подходит для предварительной оценки схемы, а не для
  итогового проекта.
- **Конструктор сети — групповая нагрузка по методу Ки/Кр:** собственную
  нагрузку узла можно задать не одним значением, а списком приёмников группы
  (установленная мощность Pн и коэффициент использования Ки каждого).
  Эффективное число приёмников `nэ = (ΣPн)²/Σ(Pн²)` и групповой коэффициент
  использования `Ки.гр = Pср/ΣPн` (где `Pср = Σ(Ки·Pн)`) считаются по точным
  формулам; расчётная нагрузка группы оценивается по прозрачной приближённой
  формуле `Кс.гр = Ки.гр + (1 − Ки.гр)/√nэ`, `Pр = Кс.гр·ΣPн` — а не по
  табличным значениям Кр(nэ, Ки.гр) из РТМ 36.18.32.4-92 / СП 256, которые
  требуют справочной таблицы и здесь не воспроизводятся. При одном приёмнике
  в группе Кс.гр = 1 (Pр = ΣPн, единственный приёмник может потребовать полную
  установленную мощность); при большом числе приёмников Кс.гр → Ки.гр (пики
  большой группы усредняются). Для точного проектирования используйте
  табличный метод.
- **Конструктор сети — компенсация реактивной мощности:** для узла сети
  переменного тока (не DC) фактический cosφ узла `cosφ1 = P/√(P²+Q²)`
  (с учётом собственной нагрузки и дочерних узлов) сравнивается с целевым
  `cosφ2`, заданным для узла (по умолчанию 0,95 — типичный диапазон,
  который сетевые организации обычно требуют от промышленных потребителей,
  0,92–0,95; значение можно изменить). Если `cosφ1 < cosφ2`, рассчитывается
  необходимая мощность конденсаторной батареи по стандартной формуле
  `Qc = P·(tgφ1 − tgφ2)`, а также реактивная и полная мощность узла после
  компенсации (`Q2 = P·tgφ2`, `S2 = √(P²+Q2²)`). Автомат и кабель узла при
  этом подбираются по фактическому (не скомпенсированному) току — это только
  рекомендация по дополнительному оборудованию, а не изменение остального
  расчёта.

## Структура проекта

```
index.html           разметка приложения
css/styles.css       стили
js/calculations.js   формулы расчёта мощности и тока (без UI)
js/tables.js         справочные таблицы кабелей/автоматов и логика подбора
js/network.js        расчёт дерева узлов сети (конструктор сети)
js/demandFactor.js   групповая нагрузка узла по методу Ки/Кр (nэ, Ки.гр, Кс.гр)
js/reactiveCompensation.js  рекомендация по компенсации реактивной мощности (Qc, cosφ)
js/history.js        хранение истории расчётов в localStorage
js/networkStorage.js хранение схемы сети конструктора в localStorage
js/schemeLayout.js   геометрия однолинейной схемы по дереву узлов
js/schemeSheet.js    оформление листа: формат, рамка, основная надпись
js/exportDxf.js      сериализация листа схемы в DXF (R12/AC1009)
js/exportPdf.js      сборка PDF с листом схемы (через холст)
js/format.js         форматирование чисел и единиц измерения
js/app.js            связывание UI с расчётной логикой
tests/               модульные тесты (Node.js test runner)
```

## Тесты

Модульные тесты для расчётной логики (`js/calculations.js`,
`js/tables.js`) на встроенном тест-раннере Node.js — внешние зависимости
не требуются:

```bash
npm test
```
